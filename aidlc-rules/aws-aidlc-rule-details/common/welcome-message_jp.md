# 👋 AI-DLC（AI駆動開発ライフサイクル）へようこそ！ 👋

お客様の特定のニーズにインテリジェントに適応する適応型ソフトウェア開発ワークフローをご案内します。

## AI-DLCとは？

AI-DLCは、プロジェクトのニーズに適応する構造化されながらも柔軟なソフトウェア開発プロセスです。次のような経験豊富なソフトウェアアーキテクトがいると考えてください：

- 必要に応じて**要件を分析**し、明確化の質問をする
- 複雑さとリスクに基づいて**最適なアプローチを計画**
- シンプルな変更のために**不必要なステップをスキップ**し、複雑なプロジェクトには包括的なカバレッジを提供
- 決定と根拠の完全な記録を持つように**すべてを文書化**
- 明確なチェックポイントと承認ゲートを使用して**各フェーズを通じてガイド**

### フェーズの詳細：

**INCEPTIONフェーズ** - *計画とアプリケーション設計*
- **目的**：何を構築するか、なぜ構築するかを決定
- **活動**：要件の理解、既存のコードの分析（ある場合）、アプローチの計画
- **出力**：明確な要件、実行計画、並行開発のための作業単位の数の決定
- **あなたの役割**：質問に答え、計画を確認し、方向を承認

**CONSTRUCTIONフェーズ** - *詳細設計、実装、テスト*
- **目的**：どのように構築するかを決定
- **活動**：詳細設計（必要な場合）、コード生成、包括的なテスト
- **出力**：動作するコード、テスト、ビルド指示
- **あなたの役割**：設計を確認し、実装計画を承認し、結果を検証

**OPERATIONSフェーズ** - *デプロイメントとモニタリング（将来）*
- **目的**：どのようにデプロイして実行するか
- **ステータス**：将来のデプロイメントとモニタリングワークフローのプレースホルダー
- **現在の状態**：ビルドとテストのアクティビティはCONSTRUCTIONフェーズで処理

## 主要原則：

- ⚡ **完全に適応的**：各ステージはニーズに基づいて独立して評価
- 🎯 **効率的**：シンプルな変更は本質的なステージのみを実行
- 📋 **包括的**：複雑な変更はすべての保護措置を持つ完全な処理を受ける
- 🔍 **透明**：作業が始まる前に実行計画を確認して承認
- 📝 **文書化**：すべての決定と変更の完全な監査証跡
- 🎛️ **ユーザーコントロール**：ステージの包含または除外を要求可能

## 次に何が起こるか：

1. **ワークスペースを分析します**：これが新しいプロジェクトか既存のプロジェクトかを理解
2. **要件を収集します**：必要に応じて明確化の質問をする
3. **実行計画を作成します**：どのステージを実行するかとその理由を示す計画を表示
4. **確認して承認します**：計画を確認（または変更を要求）
5. **計画を実行します**：各主要ステージでチェックポイントを持って実行
6. **動作するコードを取得します**：完全なドキュメントとテストを含む

AI-DLCプロセスは以下に適応します：
- 📋 意図の明確さと複雑さ
- 🔍 既存のコードベースの状態
- 🎯 変更のスコープと影響
- ⚡ リスクと品質要件
