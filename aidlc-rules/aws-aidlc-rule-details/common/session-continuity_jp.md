# セッション継続テンプレート

## お帰りなさいプロンプトテンプレート
ユーザーが既存のAI-DLCプロジェクトでの作業を続けるために戻ってきた場合、このプロンプトを提示します：

```markdown
**お帰りなさい！進行中の既存のAI-DLCプロジェクトがあります。**

aidlc-state.mdに基づく現在のステータス：
- **プロジェクト**：[project-name]
- **現在のフェーズ**：[INCEPTION/CONSTRUCTION/OPERATIONS]
- **現在のステージ**：[ステージ名]
- **最後に完了**：[最後に完了したステップ]
- **次のステップ**：[次に作業するステップ]

**今日は何に取り組みますか？**

A) 中断したところから続ける（[次のステップの説明]）
B) 以前のステージを確認する（[利用可能なステージを表示]）

[Answer]: 
```

## 必須：セッション継続の指示
1. 既存のプロジェクトを検出したときは**常にaidlc-state.mdを最初に読み取る**
2. ワークフローファイルから**現在のステータスを解析**してプロンプトに入力
3. **必須：以前のステージの成果物を読み込む** - ステージを再開する前に、以前のステージからのすべての関連成果物を自動的に読み取る：
   - **リバースエンジニアリング**：architecture.md、code-structure.md、api-documentation.mdを読み取る
   - **要件分析**：requirements.md、requirement-verification-questions.mdを読み取る
   - **ユーザーストーリー**：stories.md、personas.md、story-generation-plan.mdを読み取る
   - **アプリケーション設計**：アプリケーション設計成果物（components.md、component-methods.md、services.md）を読み取る
   - **設計（ユニット）**：unit-of-work.md、unit-of-work-dependency.md、unit-of-work-story-map.mdを読み取る
   - **ユニットごとの設計**：functional-design.md、nfr-requirements.md、nfr-design.md、infrastructure-design.mdを読み取る
   - **コードステージ**：すべてのコードファイル、計画、およびすべての以前の成果物を読み取る
4. **ステージごとのスマートなコンテキスト読み込み**：
   - **初期ステージ（ワークスペース検出、リバースエンジニアリング）**：ワークスペース分析を読み込む
   - **要件/ストーリー**：リバースエンジニアリング+要件成果物を読み込む
   - **設計ステージ**：要件+ストーリー+アーキテクチャ+設計成果物を読み込む
   - **コードステージ**：すべての成果物+既存のコードファイルを読み込む
5. アーキテクチャの選択と現在のフェーズに基づいて**オプションを適応**
6. 一般的な説明ではなく**具体的な次のステップを表示**
7. タイムスタンプ付きで**継続プロンプトをaudit.mdに記録**
8. **コンテキストサマリー**：成果物を読み込んだ後、ユーザーの認識のために何が読み込まれたかの簡単なサマリーを提供
9. **質問する**：明確化またはユーザーフィードバックの質問は、常に.mdファイルに配置してください。チャットセッションで複数選択の質問をインラインで配置しないでください。

## エラー処理
セッション再開中に成果物が欠落または破損している場合は、復旧手順のガイダンスについて[error-handling.md](error-handling.md)を参照してください。
